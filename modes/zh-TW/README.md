# career-ops — 繁體中文模式 (`modes/zh-TW/`)

此目錄收錄 career-ops 主要模式的繁體中文翻譯，適用於鎖定**台灣**市場、或處理繁體中文職缺描述的求職者。

> ⚠️ **這是台灣專屬的市場模式，不是通用的繁體中文模式。** 底下的法規、福利與求職平台全部以台灣為準。
> 香港的僱傭權益由《僱傭條例》（第 57 章）規範，與台灣《勞動基準法》完全不同——本模式的勞動法條款**不適用於香港**。
> 香港使用者若只需要繁體中文的行文，可以用這裡的散文與詞彙，但**必須忽略台灣市場表**（`_shared.md` 的「台灣市場」一節）。

## 何時使用這些模式？

符合以下任一條件，就用 `modes/zh-TW/`：

- 你主要投遞的是**繁體中文職缺描述**（104、1111、CakeResume、Yourator、Meet.jobs，或公司官網的中文版）。
- 你的**履歷**是中文，或你需要在中英文履歷之間切換。
- 你需要寫出**自然的台灣科技業中文**（求職信、LinkedIn 開發信），而不是機器翻譯腔。
- 你需要處理**台灣勞動市場特有的契約與福利條款**：勞保、健保、勞退（雇主提繳 6%）、年終獎金、員工分紅／RSU、試用期、競業禁止（勞基法第 9-1 條）、責任制（第 84-1 條）、預告期間（第 16 條）、特休（第 38 條）、資遣費、二代健保補充保費、月薪 × 14 的年薪換算等。

如果你的職缺描述大多是英文的（即使公司在台灣），建議繼續用 `modes/` 底下的預設英文模式。

> **繁體 vs 簡體：** 若你鎖定的是中國大陸市場（五險一金、落戶、996、拉勾／BOSS直聘），請改用 [`modes/zh/`](../zh/)。兩者的差別**不只是字體**，還包含勞動法規、福利結構與求職平台。

## 如何啟用？

### 方式 1：單次會談臨時啟用

在會談一開始，明確告訴 AI 助理：

> 「從現在開始使用 `modes/zh-TW/` 底下的繁體中文模式。」

或者：

> 「請用 `modes/zh-TW/_shared.md` 和 `modes/zh-TW/oferta.md` 來評估這個中文職缺。」

### 方式 2：在個人設定中永久指定

在 `config/profile.yml` 加上語言偏好，例如：

```yaml
language:
  output: zh-TW           # 對外文本的語言（報告、求職信、表單回答）
  modes_dir: modes/zh-TW  # 市場詞彙與評估規則
```

這兩個是**互相獨立**的軸（見 `AGENTS.md` 的 "Output Language vs Market Modes"）：
`modes_dir` 只提供台灣的市場脈絡，`output` 才決定產出文字的語言。若只設 `modes_dir` 而不設 `output`，
助理會載入台灣規則、但仍以預設語言（`en`）產出文字。

第一次會談時提醒助理遵循這兩個欄位即可。

## 翻譯了哪些檔案？

首批翻譯涵蓋四個最高頻、最高價值的模式：

| 檔名 | 對應原檔 | 用途 |
|---|---|---|
| `_shared.md` | `modes/_shared.md` | 共用脈絡、個人檔案偵測、全域規則，以及台灣市場特性 |
| `oferta.md` | `modes/oferta.md` | 完整職缺評估（Block A–F ＋ G 真實性查核） |
| `apply.md` | `modes/apply.md` | 網頁表單填寫的即時助理 |
| `pipeline.md` | `modes/pipeline.md` | URL 收件匣／職缺蒐集的第二大腦 |

其餘工具型模式（`scan`、`batch`、`pdf`、`tracker` 等）仍使用預設的英文版，因為它們主要由命令列參數、系統路徑與設定指令構成，保持語言一致有助於工具穩定運作。

## 常用詞彙對照

為維持語氣一致，修改或擴充模式時請遵循下列對照表：

| 英文 | 繁體中文（本專案規範）| 說明 |
|---|---|---|
| Job posting | 職缺描述 / 職缺 | 通常簡稱 JD |
| Application | 投遞 / 應徵 | |
| Cover letter | 求職信 | |
| Resume / CV | 履歷 | |
| Salary / Compensation | 薪資 / 整體報酬 | 台灣常以「月薪 × 14」換算年薪 |
| Interview | 面試 | |
| Hiring manager | 用人主管 | |
| Recruiter | 招募人員 / HR | |
| AI | AI / 人工智慧 | |
| Requirements | 職務需求 / 應徵條件 | |
| Notice period | 預告期間 | 勞基法第 16 條，依年資 10–30 日 |
| Probation | 試用期 | 勞基法未明定，實務多為 1–3 個月 |
| Non-compete | 競業禁止 | 勞基法第 9-1 條：**未給補償者無效** |
| Social insurance | 勞保 / 健保 / 勞退 | 雇主須提繳薪資 6% 至勞退個人專戶 |
| Year-end bonus | 年終獎金 | 台灣慣例 1–2 個月，**非法定保證** |
| Stock options / RSU | 員工認股權 / 限制型股票 | 本土公司另有「員工酬勞」（分紅）|
| Overtime / responsibility system | 加班 / 責任制 | 「責任制」須符合勞基法第 84-1 條並經核備，多數職務其實不適用 |
| Severance | 資遣費 | 勞退新制：每滿一年發 0.5 個月，上限 6 個月 |
| Supplementary premium | 二代健保補充保費 | 獎金超過投保薪資 4 倍的部分需扣繳 |
